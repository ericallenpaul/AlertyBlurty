# Security Policy

## Supported versions

Security fixes are applied to the latest release on `main`.

| Version | Supported |
| ------- | --------- |
| latest  | yes       |
| older   | no        |

## Reporting a vulnerability

Please do not open public issues for security reports.

1. Email: `security@alertblurty.local`
2. Include:
   - Affected version/commit
   - Reproduction steps
   - Impact assessment
   - Suggested mitigation (if available)
3. You will receive an acknowledgement within 2 business days.
4. We will provide remediation status updates until closure.

## Disclosure process

- We validate and triage reports.
- We prepare a fix and release notes.
- We coordinate responsible disclosure timing with the reporter when possible.
